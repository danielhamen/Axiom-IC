# `axiom-ic/docs/versions`

This folder contains the release notes for all versions

## Style Guide

Each release shall follow the format: `( VA.B.C.D )` and realized in the folder format `vA-B-C-D` where:
- A: Major Update (breaking such as syntax change)
- B: Minor Update (non-breaking such as new opcode)
- C: Patch Update (such as bug fix)
- D: Release Type (Pre-Alpha, Alpha, Beta, Release Candidate, RTM, Production)

The release type is to be coded as follows:

|Readable|Coded|Description|
|-|-|-|
|Pre Alpha|pre|Requirement analysis and complete feature & goal outline|
|Alpha|a||
|Beta|b||
|Release Candidate|rc||
|RTM|rtm||
|Production|pro||
